## Establecimiento del proceso productivo
La metodología que se va a llevar a cabo será SCRUM. El sistema inicialmente será planteado con base a las visitas técnicas que se le hagan al cliente y posteriormente se va a tener que plantear un plan de acción de desarrollo con un diagrama de Gantt, junto al gantt se va a tener que hacer el desglose de tareas que involucra cada uno de los módulos con sus estimados de tiempo individuales los cuales serán separados en sprints de una semana cada uno.
### 1. Planificación del Sprint
- Se definen las **tareas a desarrollar** en el sprint.
- Se asignan responsabilidades y estimaciones de tiempo.
- Se establecen los objetivos y prioridades del sprint.
### 2. Desarrollo y Construcción
- Implementación de **backend (Node.js, MariaDB)** y **frontend (React)**.
- Desarrollo modular con pruebas unitarias e integración.
- Uso de **Git** para control de versiones y revisión de código.
### 3. Pruebas y Verificación
- Ejecución de pruebas unitarias y de integración.
- Evaluación de **rendimiento, seguridad y funcionalidad**.
- Corrección de errores detectados en pruebas.
### 4. Despliegue y Mantenimiento
- Implementación en un entorno de **staging** para validación.
- Despliegue en producción utilizando **CI/CD**.
- Monitoreo y mantenimiento con actualizaciones periódicas.

### Sistema para SCRUM, versionamiento y manejo de pipelines
#### 1. Azure DevOps Services
Azure DevOps es una solución integral de Microsoft que proporciona repositorios Git para control de versiones, Azure Pipelines para CI/CD y Azure Boards para la gestión ágil de proyectos con soporte para Scrum.
- **Plan Básico**: Incluye Azure Repos, Azure Pipelines y Azure Boards. Los primeros 5 usuarios son gratuitos; a partir del sexto usuario, el costo es de $6 USD por usuario al mes.
    - **Costo anual para 6 usuarios**: $6 USD/usuario/mes × 1 usuario × 12 meses = **$72 USD**
- **Azure Pipelines adicionales**: Incluye 1,800 minutos al mes con 1 trabajo paralelo gratuito. Trabajos adicionales tienen un costo de $40 USD por trabajo paralelo al mes.
    - **Costo anual por trabajo adicional**: $40 USD/mes × 12 meses = **$480 USD**
#### 2. GitScrum
GitScrum es una herramienta diseñada para startups, agencias y desarrolladores que integra control de versiones con Git y gestión de proyectos ágil utilizando Scrum.
- **Plan Estándar**: Ofrece funcionalidades completas a un costo de $24.99 USD al mes.
    - **Costo anual**: $24.99 USD/mes × 12 meses = **$299.88 USD**
#### 3. Jira Software con Bitbucket
Atlassian ofrece Jira para la gestión ágil de proyectos y Bitbucket como plataforma de control de versiones y CI/CD.
- **Jira Software**:
    - **Plan Estándar**: $7.75 USD por usuario al mes.
        - **Costo anual para 6 usuarios**: $7.75 USD/usuario/mes × 6 usuarios × 12 meses = **$558 USD**
- **Bitbucket**:
    - **Plan Estándar**: $3 USD por usuario al mes.
        - **Costo anual para 6 usuarios**: $3 USD/usuario/mes × 6 usuarios × 12 meses = **$216 USD**
**Costo total anual para 6 usuarios**: $558 USD (Jira) + $216 USD (Bitbucket) = **$774 USD**

## Manuales de Procesos Productivos
Los manuales de procesos productivos servirán como guía para el desarrollo, implementación y mantenimiento del sistema de administración de inventarios.
### 1. Desarrollo de Software
#### 1.1 Configuración del Entorno de Desarrollo
1. Instalar **Node.js**, **MariaDB** y **Docker**.
2. Clonar el repositorio desde **Git**.
3. Instalar dependencias con `npm install`.
4. Configurar variables de entorno en un archivo **.env**.
#### 1.2 Desarrollo Backend
1. Crear estructura de API en **Node.js** con **Express**.
2. Definir modelos y relaciones en **Sequelize**.
3. Implementar controladores y rutas RESTful.
4. Configurar validaciones y seguridad.
#### 1.3 Desarrollo Frontend
1. Crear proyecto con **React**.
2. Diseñar componentes reutilizables.
3. Implementar Redux o Context API para manejo de estado.
4. Consumir API backend y manejar respuestas.

### 2. Gestión del Proyecto con Scrum
#### 2.1 Inicio del Sprint
1. Definir backlog con tareas específicas.
2. Asignar responsabilidades al equipo.
3. Establecer objetivos para el sprint.
#### 2.2 Desarrollo
1. Implementar funcionalidades según prioridad.
2. Hacer revisiones diarias (**Daily Stand-ups**).
3. Realizar **Code Reviews** antes de integrar cambios.
#### 2.3 Cierre del Sprint
1. Ejecutar pruebas unitarias e integración.
2. Desplegar en entorno de pruebas.
3. Obtener retroalimentación y planificar ajustes.
### 3. Despliegue y Mantenimiento
#### 3.1 Despliegue en Producción
1. Configurar servidor y base de datos en la nube.
2. Implementar CI/CD con **GitHub Actions / Azure Pipelines**.
3. Realizar pruebas finales antes del lanzamiento.
#### 3.2 Mantenimiento
1. Monitorear rendimiento del sistema.
2. Aplicar parches de seguridad mensualmente.
3. Programar respaldo diario de la base de datos.
## Medidas de Verificación de Calidad
Para asegurar el correcto funcionamiento del sistema de administración de inventarios, se implementarán las siguientes medidas de verificación de calidad:
### 1. Pruebas de Software
#### 1.1 Pruebas Unitarias
- Se probarán funciones y módulos individuales del backend y frontend.
- Se utilizarán herramientas como **Jest** o **Mocha**.
#### 1.2 Pruebas de Integración
- Validación de comunicación entre el frontend, backend y base de datos.
- Simulación de interacciones entre módulos.
#### 1.3 Pruebas de Carga
- Evaluación del rendimiento con múltiples usuarios simultáneos.
- Uso de herramientas como **JMeter** o **K6**.
#### 1.4 Pruebas de Seguridad
- Análisis de vulnerabilidades en API y autenticación.
- Validación de cifrado de datos y protección contra inyecciones SQL.
#### 1.5 Pruebas de Usuario
- Pruebas en un entorno controlado con usuarios reales.
- Retroalimentación para mejorar la experiencia y corregir errores.
### 2. Control de Calidad en el Proceso de Desarrollo
#### 2.1 Revisión de Código
- Implementación de **Code Reviews** antes de hacer merge en el repositorio.
- Uso de herramientas como **ESLint** y **Prettier** para mantener buenas prácticas.
#### 2.2 Uso de CI/CD (Integración y Despliegue Continuo)
- Configuración de **pipelines automáticos** para pruebas antes de cada despliegue.
- Implementación de **entornos de staging** para validaciones previas.
### 3. Monitoreo Post-Despliegue
#### 3.1 Registro de Errores y Monitoreo
- Uso de **Sentry** o **Datadog** para capturar errores en tiempo real.
- Implementación de logs en **Azure Application Insights**.
#### 3.2 Soporte y Mantenimiento
- **Actualizaciones periódicas** de seguridad y optimización.
- **Monitoreo de rendimiento** con alertas automáticas.
